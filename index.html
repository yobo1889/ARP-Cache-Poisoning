ARP Poisoning and MITM Attack Simulation

Project Overview

This project demonstrates ARP poisoning to execute Man-in-the-Middle (MITM) attacks, showcasing network protocol vulnerabilities. The attack involves manipulating the target's ARP cache to intercept, monitor, and modify network traffic, emphasizing the importance of ARP spoofing detection and network encryption for enhanced data integrity and privacy.

How to Run the Project

To run the ARP poisoning attack simulation, follow these steps:

  1. Clone the repository from GitHub.
  2. Ensure you have the required dependencies installed, including necessary networking tools and libraries.
  3. Set up a NAT-configured network with two virtual machines.
  4. Compile the source code and run the program using: ./arp
